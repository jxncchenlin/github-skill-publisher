# GitHub Skill Publisher 使用示例

## 示例1：首次发布"MD转微信"技能

### 场景
您开发了一个将Markdown转换为微信公众号格式的技能，现在要发布到GitHub。

### 操作步骤

**第1步：运行交互式向导**

```bash
python skills/github-skill-publisher/scripts/publisher.py --guide
```

**第2步：按照提示输入信息**

```
第1步：基本信息
📝 请输入技能名称: md-to-wechat
📝 请输入技能描述: 将Markdown转换为微信公众号格式，完美适配个人订阅号
📝 请输入版本号（默认 v1.0.0）: 1.0.0
📝 请输入作者名称（默认 YourName）: YourName
📝 选择开源协议:
  1. MIT
  2. Apache-2.0
  3. GPL-3.0
📝 请选择（默认 1-MIT）: 1

继续吗？(Y/n): Y

第2步：功能特性
📝 请列出3-5个核心功能（每行一个，空行结束）:
  1. 三种主题风格
  2. 一键转换
  3. 批量处理
  4. 支持代码高亮
```

**第3步：查看生成的文件**

```
md-to-wechat/
├── README.md                    # 自动生成，完整的项目说明
├── LICENSE                      # MIT协议
├── .gitignore                   # Git配置
├── SKILL.md                     # 技能详细文档
├── CHANGELOG.md                 # 版本日志
├── md-to-wechat.skill          # 技能包（示例）
├── github-publish-guide.md     # 发布指南
└── .github/
    ├── ISSUE_TEMPLATE/
    │   ├── bug_report.md       # Bug报告模板
    │   └── feature_request.md  # 功能请求模板
    └── pull_request_template.md
```

**第4步：替换实际的技能文件**

将您开发的实际技能文件复制到生成的目录：
```bash
cp -r your-original-skill/* md-to-wechat/
```

**第5步：创建GitHub仓库**

1. 访问 https://github.com/new
2. Repository name: `md-to-wechat`
3. Description: `将Markdown转换为微信公众号格式，完美适配个人订阅号`
4. 选择 Public
5. 不要勾选 "Initialize this repository with a README"
6. 点击 "Create repository"

**第6步：上传文件**

最简单的方式 - 拖拽上传：
1. 在仓库页面点击 "uploading an existing file"
2. 将 `md-to-wechat/` 文件夹中的所有文件拖拽到浏览器
3. Commit message: `Initial release`
4. 点击 "Commit changes"

**第7步：创建Release**

1. 在仓库页面点击 "Releases" → "Draft a new release"
2. Choose a tag: 输入 `v1.0.0`，点击空白处
3. Release title: `MD转微信公众号技能 v1.0.0`
4. Description:
   ```
   将Markdown转换为微信公众号格式，完美适配个人订阅号。特性：三种主题、一键转换、批量处理。下载md-to-wechat.skill，在WorkBuddy中安装使用。
   ```
5. Attach binaries: 上传 `md-to-wechat.skill`
6. 点击 "Publish release"

完成！您的技能已成功发布到GitHub！

## 示例2：命令行批量发布多个技能

### 场景
您有多个技能需要一次性发布。

### 操作步骤

**创建批量配置文件** `batch-skills.json`:

```json
{
  "skills": [
    {
      "name": "md-to-wechat",
      "desc": "将Markdown转换为微信公众号格式",
      "version": "1.0.0",
      "author": "YourName",
      "license": "MIT",
      "features": [
        "三种主题风格",
        "一键转换",
        "批量处理"
      ]
    },
    {
      "name": "image-optimizer",
      "desc": "图片压缩优化工具",
      "version": "1.0.0",
      "author": "YourName",
      "license": "MIT",
      "features": [
        "智能压缩",
        "批量处理",
        "支持多种格式"
      ]
    }
  ]
}
```

**运行批量发布脚本**:

```python
#!/usr/bin/env python3
import json
import sys
from pathlib import Path

# 导入发布工具
sys.path.insert(0, str(Path(__file__).parent))
from publisher import GitHubSkillPublisher

# 加载配置
with open('batch-skills.json', 'r', encoding='utf-8') as f:
    config = json.load(f)

# 批量发布
for skill_config in config['skills']:
    print(f"\n发布技能: {skill_config['name']}\n")
    print("=" * 50)

    publisher = GitHubSkillPublisher(
        name=skill_config['name'],
        desc=skill_config['desc'],
        version=skill_config['version'],
        author=skill_config['author'],
        license_type=skill_config['license'],
        features=skill_config.get('features', [])
    )

    publisher.generate_all()

    print(f"\n✅ {skill_config['name']} 发布文件已生成\n")
```

运行：
```bash
python batch-publish.py
```

## 示例4：使用自定义README模板

### 场景
您希望使用自定义的README模板。

### 操作步骤

**第1步：创建自定义模板** `custom-readme.md`:

```markdown
# {{name}}

{{badges}}

> {{desc}}

## 🎯 快速开始

{{install_steps}}

## 📚 文档

- [安装指南](#安装指南)
- [使用说明](#使用说明)
- [API文档](#api文档)

## 🤝 贡献

欢迎提交Issue和Pull Request！

## 📝 许可证

{{license_type}} License
```

**第2步：使用自定义模板发布**

修改 `publisher.py` 中的 `generate_readme()` 方法，添加模板支持：

```python
def generate_readme(self, template_path=None):
    """生成README.md"""
    if template_path and Path(template_path).exists():
        with open(template_path, 'r', encoding='utf-8') as f:
            template = f.read()

        # 替换占位符
        content = template.replace('{{name}}', self.name)
        content = content.replace('{{desc}}', self.desc)
        # ... 更多替换
    else:
        # 使用默认模板
        content = self._default_readme_content()

    readme_path = self.output_dir / "README.md"
    readme_path.write_text(content, encoding='utf-8')
```

## 示例5：集成到CI/CD流程

### 场景
希望自动化发布流程，使用GitHub Actions。

### 创建 `.github/workflows/release.yml`:

```yaml
name: Release

on:
  push:
    tags:
      - 'v*'

jobs:
  release:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v3

      - name: Generate release files
        run: |
          python skills/github-skill-publisher/scripts/publisher.py \
            --name "md-to-wechat" \
            --desc "将Markdown转换为微信公众号格式" \
            --version "${GITHUB_REF#refs/tags/v}" \
            --author "GitHub Actions"

      - name: Create Release
        uses: softprops/action-gh-release@v1
        with:
          files: |
            md-to-wechat.skill
          body_path: RELEASE.md
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

**使用**:
```bash
git tag v1.0.0
git push origin v1.0.0
```

GitHub Actions会自动创建Release！

## 技巧提示

1. **保存配置**: 将常用的参数保存到配置文件中，方便重复使用
2. **版本管理**: 遵循语义化版本规范（Semantic Versioning）
3. **文档完善**: 发布前仔细检查生成的文档，确保信息准确
4. **测试验证**: 发布前在本地测试技能包是否正常工作
5. **持续改进**: 根据用户反馈不断改进文档和功能

## 故障排除

### 问题1：生成的README格式不对

**解决**: 编辑生成的README.md，使用标准Markdown格式

### 问题2：技能包无法安装

**解决**: 确保技能包文件结构正确，替换示例文件为实际文件

### 问题3：GitHub Release创建失败

**解决**: 检查tag名称格式是否正确（如v1.0.0）
